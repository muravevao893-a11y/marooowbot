# maroow giveaway bot — chance mode

Главная авто-механика теперь не таймерный рандом под постом, а **шанс за комментарий**.

- бот видит новый пост канала в discussion-чате;
- пишет под ним сообщение в стиле “Кто хочет мишку?”;
- каждый комментарий зарегистрированного подписчика получает шанс `CHANCE_DROP_PERCENT`, по умолчанию 3%;
- если шанс выпал, бот отвечает прямо на комментарий победителя;
- в ответе есть кнопка **🎁 Забрать мишку**;
- кнопку может нажать только победитель;
- после нажатия бот отправляет Telegram Gift по `AUTO_DROP_GIFT_ID`;
- ручные розыгрыши через `/admin` остались.

В `.env` главное:

```env
AUTO_DROPS_ENABLED=true
AUTO_DROP_PRIZE=мишка
AUTO_DROP_GIFT_ID=your_gift_id
AUTO_DROP_WINNERS=1
AUTO_DROP_DURATION_SECONDS=86400
CHANCE_DROP_PERCENT=3
```

# maroow giveaway bot

Бот для Telegram-канала: авто-розыгрыши под каждым новым постом + ручные розыгрыши с таймером, счетчиком участников, профилями и отправкой Telegram Gifts.

## Что умеет

- `/start` — регистрация профиля пользователя.
- Авто-дроп под каждым новым постом канала через discussion-группу.
- Сбор участников из комментариев к посту.
- Ручное создание розыгрышей из админки бота.
- Inline-кнопка участия со счетчиком.
- Проверка регистрации пользователя.
- Проверка подписки на канал.
- Выбор победителей через `secrets.SystemRandom`.
- Отправка Telegram Gift через `sendGift`, если указан `gift_id`.
- Если `gift_id` не указан или отправка упала — победитель помечается как `manual_required/failed`, чтобы подарок можно было выдать руками.
- Бан-лист, статистика, список доступных подарков.

## Важно перед запуском

1. Создай бота в @BotFather.
2. Добавь бота админом в канал.
3. Добавь бота админом в discussion-группу канала.
4. В @BotFather отключи privacy mode или оставь бота админом с правом читать сообщения, иначе он не будет видеть обычные комментарии.
5. Узнай numeric ID канала и чата обсуждения. Удобно через временного debug-бота или `getUpdates`.
6. Заполни `.env` по примеру `.env.example`.

## Запуск локально

```bash
cp .env.example .env
# отредактируй .env

docker compose up -d postgres redis
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m app.main
```

По умолчанию бот стартует в polling mode. Для webhook укажи `WEBHOOK_URL`, например `https://your-domain.com`, и прокинь HTTPS через nginx/Caddy.

## Админка

Открой бота в личке админом и напиши:

```text
/admin
```

Там будут кнопки:

- создать розыгрыш;
- авто-дропы;
- подарки;
- статистика;
- бан/разбан.

## Как работает авто-дроп

Когда ты публикуешь пост в канал, Telegram автоматически пересылает его в привязанную discussion-группу. Бот ловит это сообщение, создает розыгрыш, отвечает под ним красивым комментом и начинает собирать участников из комментариев этого треда.

Учитываются только пользователи, которые:

- зарегистрированы через `/start`;
- не забанены;
- подписаны на канал, если включен `REQUIRE_SUBSCRIPTION=true`;
- оставили комментарий под нужным постом.

Один пользователь = один шанс.

## Telegram Gifts

Чтобы бот автоматически отправлял подарок, в розыгрыше должен быть указан `gift_id`. Список доступных подарков можно посмотреть через `/gifts` или кнопку в админке.

Если Telegram Gift недоступен, закончился или у бота не хватает Stars, бот не ломается: он сохранит ошибку и отметит выдачу как требующую ручной проверки.

## Команды

Пользовательские:

```text
/start
/profile
/rules
```

```text

/gifts
/cancel
/ban <telegram_id>
/unban <telegram_id>
```
